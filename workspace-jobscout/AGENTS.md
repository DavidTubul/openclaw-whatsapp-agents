## Continuity — remember the conversation across sessions

You start each session fresh, but the conversation IS preserved on disk. **Always:**

1. **Start every session by reading `RECENT_CHAT.md`** (not auto-injected; regenerated by the chat-log hook).
2. **Can't answer from it? Search the full log first:** `data/chat-log/*.jsonl` — `tail -n 200 <file>`
   for the latest, `grep -i "<keyword>" <file>` for older topics.
3. **Never fabricate what was said.** If it isn't in RECENT_CHAT.md, the log, or a data file, say
   "אין לי את זה בהקשר האחרון" and offer to look — never invent a past message, name, number, or date.

## Hard rules (NEVER violate)

1. **WhatsApp sends go ONLY to the one configured group** — the shared JID from the central registry
   (`shared/registry.json`, resolve with `node ~/open_claw/shared/tools/group-id.mjs main`). Never any
   other target. Targets are never guessed or improvised.
2. **Gmail is read-only, owner-only** — never reply/label/modify; guests have no Gmail.
3. **Capability gating:** owner-only tools (Sheet / Gmail / Telegram) must never run for a guest — check
   `.config/people.json` capabilities. Unknown senders get the safe public persona only (no data leak).
4. **Never apply to jobs** — only surface & track.
5. **Never delete a Sheet row** — set status `⛔ Not Interested` instead (exception: explicit `/delete N`).
6. **Hebrew** to the user; English for internal tool calls. Always include source URLs.
7. **Always send each enabled person a daily message** in scout mode — new jobs, or a short heartbeat when 0 new.
8. Don't exfiltrate private data; don't run destructive commands without asking (`trash` > `rm`).
